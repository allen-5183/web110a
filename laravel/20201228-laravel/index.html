    本例: 以磁碟代號 D: 安裝。

  • 安裝完成，會在 d:\composer 目錄下，產生 composer.phar。

  • 執行 composer 時, 可以使用 php composer.phar 執行或將它建立成 composer.bat 來使用，編輯 composer.bat 內容如下:

    @php "%~dp0composer.phar" %*

  • 追加 composer 安裝套件全域環境變數:
    PATH= ;d:\composer

  • 追加所有專案共享的全局目錄
    COMPOSER_HOME d:\composer

  • 把 composer 套用到 PHP 專案上

    初始化:

    cd  /路徑/專案   # 例: D:\xampp\htdocs\laravel
composer  init  # name、description、authors、minimum-stability、license、require、require-dev

    • Package name(vendor/name)

      • 套件名稱，格式為: “vendor_name/project_name”
      • 作為 composer require 套件名稱 時候所使用的套件名稱

        例: [allen/laravel]

    • Description[]

      • 簡短的套件描述
      • 儘量在一行內結束
      • 一定要輸入

        例: This is a Laravel Project

    • Authors[, n to skip]

      • 套件開發者資訊
      • 格式: “name “
        例: Allen Tai <allen5183@gmail.com>
      • 可以有多個 author (需手動加入)
      • 預設抓取 ~/.gitconfig 的設定。

    • minimum-stability

      • 套件穩定度的說明與指定
      • 只能用在 root-package
      • 可設定的值: (由最穩定到最不穩定的順序)
        • stable(預設)
        • RC
        • beta
        • alpha
        • dev

          相依的套件之穩定度不可低於設定值。
          例: dev

    • Package Type

      • 用來標示套件性質，無實質功能
      • 一般常用 project, 也可以自己設定。
      • 其他配置名稱可為 library、metapackage、composer-plugin

    • license

      • 版權說明，若為私有專案可用 proprietary
      • 可參考 SPDX 的 identifier 來設定
        • Apache-2.0
        • BSD-2-Clause
        • MIT
        • …

          例: MIT

    • Define your dependencies.

      • Would you like to define your dependencies (require) interactively [yes]?
        Search for a package: log

        選擇 [0]psr/log

        Enter the version constraint to require (or leave blank to use the latest version): *

        • require
          • 用來標示正式環境依賴的套件以及版本號
            格式: “vendor/package-name” : “version”
          • 指定專案一定要安裝的套件或平台環境
          • 如果現存的套件版本不符合 version 條件的話，就不會安裝任何套件。

      • Would you like to define your dev dependencies (require-dev) interactively [yes]?
        Search for a package: phpunit

        選擇 [0]psr/log

        Enter the version constraint to require (or leave blank to use the latest version): >=3.7.0

        • require-dev
          • 用來標示開發環境依賴的套件以及版本號
          • 格式同 require
          • 主要指定開發用的套件
          • 只能用在 root-package

    • 產生 `composer.json

      {
  "name": "allen/tai",
  "description": "My First Application",
    "require":{
    "psr/log": "*"
  },
  "require-dev" :{
    "phpunit/phpunit":">=3.7.0"
  },
  "license" : "MIT",
  "authors" : [
    {
      "name": "Allen Tai",
      "email": "allen5183@gmail.com"
    }
  ],
  "minimum-stability": "dev"
}
      1. 所有 package 都需要 composer.json
        composer.json 的部份屬性只有在 root-package 才有作用。
      2. 只有專案負責人才做第一次 install 及之後的 update (會更動 composer.lock)
      3. 其他成員一律使用 install
      4. 第一次 install 及之後的 update 會將所有相依套件的資訊編寫到 composer.lock 上，並固定套件版本
      5. –prefer-source 從版本控制下載
      6. –prefer-dist 下載壓縮檔

  • vendor 資料夾

    • 套件預設會安裝在 vendor 資料夾
    • 自動載入的載入器與快取檔案放在
      vendor/composer 下

  • 全新安裝時之流程:

    1. composer.json # 第一次全新安裝會參考 composer.json
    2. Download list # 接下來就會去官方套件庫下載清單
    3. Download package # 找到符合版本設定的套件並下載
    4. Generate composer.lock # 產生 composer.lock 並記住每個套件在此安裝時的版本，這樣一來就不再從清單理找符合的版本
    5. Generate autoloading files # 將所有相依套件的自動載入設定加到快取檔案中

  • 再次安裝之流程:

    1. composer.lock # 直接找 composer.lock 記錄的版本
    2. Download package # 然後從快取或來源中獲得套件檔案
    3. Generate composer.lock # 將所有相依套件的自動載入設定加到快取檔案中

  • 版本控制

    • root-package 的 composer.json 及 composer.lock 要放到版本控制系統裡，這麼一來其他成員可以直接使用。
    • library 的 composer.lock 則不要放到版本控制系統中
    • vendor 資料夾不要放到版本控制系統，應該用 composer install 安裝。

  • packagist.org

    • 記錄常見的 Open Source library 資訊

    • 預設 composer 搜尋套件的地方

    • 不須審核，人人都可以註冊自己的套件

    • 不是實際存放套件內容的地方

    • 運作方式

      • composer 向 packagist 搜尋套件
      • packagist 回傳套件清單給 composer
      • composer 依照套件清單找到來源
      • composer 從來源取回套件內容

  • 自動載入

    • 當要使用到的類別，不存在時，自動載入該類別。

    • 為什麼 composer 要提供自動載入
      在 PHP 開發過程中，如果希望從外部引入一個 class，通常會使用 include 和 require 方法，去把定義這個 class 的檔案包含進來。這個在小規模開發的時候，沒什麼大問題。但在大型的開發專案中，使用這種方式會帶來一些隱含的問題：如果一個 PHP 檔案需要使用很多其它類，那麼就需要很多的 require/include 語句，這樣有可能會造成遺漏或者包含進不必要的類檔案。如果大量的檔案都需要使用其它的類，那麼要保證每個檔案都包含正確的類檔案肯定是一個噩夢， 況且 require_once 的代價很大。

      PHP5 為這個問題提供了一個解決方案，這就是類的自動裝載 (autoload) 機制。autoload 機制可以使得 PHP 程式有可能在使用類時才自動包含類檔案，而不是一開始就將所有的類檔案 include 進來，這種機制也稱為 lazy loading。

    • 優點

      • 統一自動載入的方法
      • 每個 Package 的目錄結構不見得相同
      • 讓 Package 自行提供載入方式
      • 使用類之前無需 include 或者 require
      • 使用類的時候才會 require/include 檔案，實現了 lazy loading，避免了 require/include 多餘檔案。

    • 範例: composer.json

      {
  "autoload":{
    "psr-o":{
      "Monolog\\": ["src/",  "lib/"],
      "Zend_": "library/"
    },
    "classmap":["classes/", "Something.php"],
    "files": ["src/MyLibrary/function.php"]
  }
}
      psr-0 符合 PSR-0 規範的類別檔案
      classmap 對應類別名稱的檔案
      files 非類別但需要一開始就自動載入的檔案

      小提醒:

      1. autoload 採相對於 composer.json 的路徑
      2. 每個套件只需負責自己的 autoload
      3. 在 root-package 執行:
        composer dump-autoload
        就會把所有相依套件的 autoload 屬性
        編譯到 vendor 目錄裡的快取檔案
      4. 載入 loader 之後就不用自行引用類別定義檔
      5. MVC Framework 只要在 index.php 載入 loader 即可
1.1.4.4 Windows Install 方式安裝

  • 安裝檔，下載網址

    PS: 如之前已安裝過，需升級成新版本，可使用以下指令

    composer self-update

  • 常用指令

    • 升級版本
      composer self-update

    • 版本查詢
      composer -V

    • 顯示有哪些指令和選項可用
      composer list

    • 顯示指令的詳細用法
      composer help

    • 安裝相依套件
      composer install

      • 使用時機: 剛clone專案之後，或需要安裝專案相依套件時。
      • 運行程序:
        1. 檢查 composer.json 是否存在。
        2. 不存在的話，就跑 update 產生一個 composer.json。
        3. 存在的話，就依照 lock 的套件版號來安裝。
        4. 更新 autoload.php 程式。

    • 安裝需求套件
      composer require 套件名稱

      • 使用時機: 載入新套件到專案內。
      • 運行程序:
        1. 寫入 composer.json。
        2. 開始下載指定套件，並挑選最佳解的版本號。
        3. 更新 autoload.php 程式。

    • 更新套件
      composer update

      • 使用時機: 修改composer.json 後，或需要更新套件版本時。

      • 更新 autoload.php
        composer dump-autoload

      • 使用時機: 當套件以位於 vendor 資料夾，但程式無法自動載入時。

1.1.6 建立 Laravel 專案

  1. 開啟 Terminal，切換到 xampp 的 htdocs 資料夾，輸入指令，路徑應根據您的狀況作調整:

    mkdir d:\xampp\htdocs
cd d:\xampp\htdocs

  2. 如果要建立的是最新 Laravel版本的專案，請輸入指令

    laravel new app_name

    例: laravel new ezsite

  3. 如果要建立指定 Laravel 版本的新專案，例如 5.8，需使用 composer，請輸入指令:

    composer create-project --prefer-dist laravel/laravel app_name "5.8.*"

  4. 開啟專案，確定是否有 .env 檔案，如果找不到 .env 檔，可將 .env.example 檔案改名成 .env。

    cd d:\xampp\htdocs\ezsite
dir .env

  5. 檢查專案下的 .env 內文是否有:
    APP_KEY=base64:8nRS4r2Ntkhuo+fvdov/Mxc6Je2R+84SHH0KTjSA430=

    如果沒有可以透過下列方式產生應用金鑰，開啟 Terminal，切換到新建立的專案資料夾，請輸入指令:

    php artisan key:generate

    artisan 是一 laravel 的管理工具，利用它來執行 laravel 的各種指令。

  6. 開啟專案，確定是否有 .env 檔案，如果找不到 .env 檔，可將 .env.example 檔案改名成 .env

1.1.7 確認建置是否成功

  • 請確保 Apache 服務和 MySQL 服務都有開啟

  1. 專案下，啟動專案

    cd d:\xampp\htdocs\ezsite
php artisan serve

  2. 開啟瀏覽器，比如 Google Chrome

  3. 輸入網址　http://localhost:8000/

1.1.8 設定本地化網域(非必須)

  1. 將所命名的網域指向到本機 127.0.0.1，作法是修改 hosts，這裡以 ezsite.test 為例

    nano C:\windows\system32\drivers\etc\hosts
127.0.0.1 ezsite.test # Laravel 專案

  2. 告訴此網域需要由哪個專案資料夾來接手，作法是修改 httpd-vhost.conf，路徑應根據您的狀況作調整。

    //XAMPP資料夾\apache\conf\extra\httpd-vhosts.conf
<VirtualHost *:6080>
  DocumentRoot "D:/xampp/htdocs/ezsite/public"
  ServerName ezsite.test
</VirtualHost>

  3. 要求 Apache 開啟時需讀取 httpd-vhosts.conf，作法是修改 XAMPP 資料夾\apache\conf\httpd.conf。找到 Include conf/extra/httpd-vhosts.conf 那一行，把前面的 # 註解移除。

  4. 重開 Apache 服務，訪問 http://ezsite.test:6080，看能否順利開啟。

  1. 開啟安裝 Homebrew 網站去複製指令，網址為 https://brew.sh

  2. 開啟 Terminal，貼入以下指令並執行

  3. 開啟 Terminal，輸入指令:

    brew install php@7.4 
brew link php@7.4 --force

    ==PS: 7.4 版本應替換為你所安裝 XAMPP 所採用的 PHP 版本==

1.2.5 下載 Laravel 安裝器

==如果進行這個階段遇到錯誤，請先退回到 Step4 安裝完再回來。如果跳出需要命令列開發者工具，點安裝來安裝工具==

  1. 開啟 Terminal，輸入以下指令
    composer global require "laravel/installer"

  2. 開啟 .bash_profile 以設定路徑，輸入指令:touch ~/.bash_profile;open ~/.bash_profile;，修改完整程式碼如下:

    export PATH=$HOME/.composer/vendor/bin:/Applications/XAMPP/xamppfiles/bin:/usr/local/bin:$PATH

  3. 立即讀取路徑設定，指令為 source ~/.bash_profile

1.2.6 建立 Laravel 專案

  1. 開啟 Terminal，切換到 xampp 的 htdocs 資料夾，輸入指令，路徑應根據您的狀況作調整

    cd /Applications/XAMPP/htdocs

  2. 如果要建立指定 Laravel 版本的新專案，例如 5.8，需使用 composer，請輸入指令:

    composer create-project --prefer-dist laravel/laravel app_name "5.8.*"

    2.1(可選) 如果要建立的是最新 Laravel 版本的專案，請輸入指令

    laravel new app_name

    例: laravel new ezsite

  3. 生成應用金鑰，開啟 Terminal，切換到新建立的專案資料夾，輸入指令

    php artisan key:generate

  4. 將 storage 資料夾權限全開，同樣在 Terminal，路徑在專案資料夾，輸入指令

    sudo chmod -R 777 storage

  5. 開啟專案，確定是否有 .env 檔案，如果找不到 .env 檔，可將 .env.example 檔案改名成 .env

1.2.7 確認建置是否成功

==請確保 Apache 服務和 MySQL 服務都有開啟==

  • 執行如下:

    cd /Applications/XAMPP/htdocs/ezsite
php artisan serve

  • 開啟瀏覽器，比如 Google Chrome

  • 輸入網址　http://localhost:8000/

1.2.8 設定本地化網域(非必須)

  1. 將所命名的網域指向到本機 127.0.0.1，作法是修改 hosts，這裡以 ezsite.test 為例

    //etc\hosts
127.0.0.1 ezsite.test #Laravel 專案

  2. 告訴此網域需要由哪個專案資料夾來接手，作法是修改 httpd-vhost.conf，加入以下設定，路徑應根據您的狀況作調整

    //XAMPP資料夾\etc\extra\httpd-vhosts.conf
<VirtualHost *:6080>
   DocumentRoot "/Applications/XAMPP/htdocs/ezsite"  
   ServerName ezsite.test
</VirtualHost>

  3. 要求 Apache 開啟時需讀取 httpd-vhosts.conf，作法是修改 XAMPP 資料夾 \etc\httpd.conf。找到 httpd-vhosts.conf 那一行，把前面的 # 註解移除。

  4. 重開 Apache 服務

  5. 開啟瀏覽器，訪問 http://ezsite.test:6080，看能否順利開啟

1.3 Linux 作業系統

1.3.1 安裝 Composer

  • 開啟終端機

    curl -sS https://getcomposer.org/installer  | php
php composer.phar <command>
sudo mv composer.phar  /user/local/bin/composer

    如果希望讓所有人都可以使用，就將 composer.phar 搬到共用執行目錄，例如 /usr/local/bin
    這樣就可以直接執行 composer，若不能執行，則要以 chmod 加入可執行權限。

2. 專案介紹、設定、MVC 概念

主設定檔修改

  • 修改 config/app.php，設定 timezone 時區與 locale 語系

    //config/app.php

'locale' => 'zh_TW',           # 'en'
'timezone' => 'Asia/Taipei',   # 'UTC'

隱私設定檔設定

  • 修改 .env 檔案，所設的值不可以有空格

    //.env

APP_NAME=ezsite //專案名稱
APP_ENV=local      //專案所在環境，local為本地端開發，production為正式上線環境
APP_KEY //應用金鑰，不得為空，可透過 php artisan key:generate來生成
APP_DEBUG=true     //是否開啟除錯模式，正式上線環境建議關掉
APP_URL= http://ezsite.test:6080 //這裡需須設定到專案下的 public 資料夾

DB_CONNECTION=mysql    //所要連接的資料庫
DB_HOST=127.0.0.1      //資料庫 Ip Address
DB_PORT=3306           //資料庫 port號
DB_DATABASE= ezsite    //資料庫名稱
DB_USERNAME= root      //資料庫帳號
DB_PASSWORD= 1qaz@wsx  //資料庫密碼
